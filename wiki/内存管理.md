# 内存管理

lwIP 提供两种内存管理机制：**堆内存 (heap)** 和 **内存池 (pool)**。

## 堆内存 (Dynamic Memory)

### 源文件
- `src/core/mem.c` - 堆内存实现
- `include/lwip/mem.h` - 头文件

### 关键特性

> [!tip] 特性
> - 类似标准 C 的 `malloc/free`
> - 支持线程安全（可选）
> - 适用于可变大小内存分配

### API

```c
#include "lwip/mem.h"

// 分配内存
void *mem_malloc(size_t size);

// 释放内存
void mem_free(void *mem);

// 重新分配
void *mem_realloc(void *mem, size_t size);
```

### 配置选项

```c
// lwipopts.h
#define MEM_SIZE                    4096    // 堆大小
#define MEM_ALIGNMENT               4       // 对齐要求
#define MEM_THREAD_SAFE            1        // 线程安全
#define MEM_OVERFLOW_CHECK         1        // 溢出检测
```

## 内存池 (Memory Pools)

### 源文件
- `src/core/memp.c` - 内存池实现
- `include/lwip/memp.h` - 头文件

### 关键特性

> [!tip] 特性
> - 固定大小内存块
> - 分配速度快，无碎片
> - 用于协议栈内部数据结构

### 内存池类型

| 类型 | 描述 | 用途 |
|------|------|------|
| `MEMP_PBUF` | pbuf 内存池 | 数据包缓冲 |
| `MEMP_RAW_PCB` | Raw API PCB | 原始连接 |
| `MEMP_TCP_PCB` | TCP PCB | TCP 连接 |
| `MEMP_UDP_PCB` | UDP PCB | UDP 连接 |
| `MEMP_NETBUF` | netbuf 结构 | Socket API |
| `MEMP_NETCONN` | netconn 结构 | Netconn API |

### API

```c
#include "lwip/memp.h"

// 从指定池分配
void *memp_malloc(memp_t type);

// 释放到指定池
void memp_free(memp_t type, void *mem);
```

### 配置

```c
// lwipopts.h
#define MEMP_NUM_PBUF              16
#define MEMP_NUM_RAW_PCB           4
#define MEMP_NUM_TCP_PCB           5
#define MEMP_NUM_UDP_PCB           4
```

## PBUF 内存池

pbuf 是 lwIP 中数据包管理的基础结构，通常从专用池分配：

```c
#define MEMP_NUM_PBUF              16      // PBUF 数量
#define MEMP_NUM_PBUF_POOL         32      // PBUF Pool 数量
#define PBUF_POOL_SIZE             256     // 每个池大小
```

## 内存布局

```
+------------------+ <- 内存起始
|     内存池       |  固定大小块，无碎片
+------------------+
|     堆内存       |  可变大小，动态分配
+------------------+ <- MEM_SIZE
```

## 选择建议

| 场景 | 推荐方式 |
|------|----------|
| 协议栈内部结构 | 内存池 |
| 应用层缓冲 | 内存池 (pbuf) 或堆 |
| 可变大块数据 | 堆内存 |
| 高性能需求 | 内存池 |
